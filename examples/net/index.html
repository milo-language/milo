server running router pattern keep-alive

Router + Dispatch

Routes stored in Vec<Route> with fn pointer handlers. Closure captures route table for dispatch.

Memory Safe

Move semantics, ownership tracking, no GC. Use-after-move caught at compile time.

LLVM Backend

Compiles to LLVM IR, then native code via clang. Full optimization pipeline.

Fn Pointers in Structs

Store fn(&Request): Response in struct fields. { ptr, ptr } calling convention.

Router Source

struct Route {
    method: string,
    prefix: string,
    exact: bool,
    handler: fn(&Request): Response,
}

fn main(): i32 {
    var routes: Vec<Route> = Vec.new()
    routes.push(Route { method: "GET", prefix: "/", exact: true, handler: home })
    routes.push(Route { method: "GET", prefix: "/fib/", exact: false, handler: fib })
    // ...
    serve(8080, (req: &Request) => {
        return dispatch(routes, req)
    })
}
