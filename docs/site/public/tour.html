Meet Milo.
Ten ideas, ten programs.

A language with ownership instead of a garbage collector — the compiler proves your memory is safe, then hands the code to LLVM for native speed.

no garbage collector no use-after-free let / var ownership green tasks + channels → LLVM native binary

Every snippet is a real program — edit it, then Run it. Milo compiles to JavaScript right here in your browser.

What "Run" really does

Milo's own compiler (written in TypeScript) walks each program through a fixed pipeline. The native path ends at a machine binary; a second backend emits JavaScript so the same code can run in a browser playground.

source→ lexer→ parser→ type + move checker→ HIR→ codegen→ LLVM IR→ binary

The type + move checker is where Milo earns its safety: use-after-move, storing a reference past its owner, and mutating an immutable binding are all caught here — before a single instruction is emitted. Snippet #9 shows it saying no.

Outputs shown are captured from the real compiler: milo run <file>.milo. Try your own in the in-browser playground built on Milo's JavaScript backend.
